# ネイティブ・ローカルエージェント（Claude CLI 不要）

[English](NATIVE_AGENT.md) · [한국어](NATIVE_AGENT.ko.md) · 日本語 · [简体中文](NATIVE_AGENT.zh-CN.md)

DevPrism には、ローカルの [Ollama](https://ollama.com) モデルと**直接通信する**組み込み
エージェントランタイムが含まれています。Claude Code CLI も変換プロキシも一切不要で、
完全にオフラインかつ自己完結で動作します。

## 有効化の手順

1. Ollama をインストールして起動し、**ツール呼び出し（tool calling）対応**のモデルを
   取得します:
   ```bash
   ollama pull llama3.1      # または qwen2.5, mistral-nemo, …
   ```
   （ツール呼び出し非対応の小型モデルは会話はできますがツールを使えません。）
2. DevPrism で: **設定 → プロバイダー（Provider）→「ネイティブ・ローカルエージェント
   （Claude CLI 不要）」**。
3. （任意）同じパネルで Ollama のエンドポイント／モデルを OpenAI 互換プロバイダーとして
   設定します。設定しない場合、ランタイムは既定の `http://localhost:11434` と最初に
   インストールされたモデルを使用します。

トグルがオンのときチャットはネイティブランタイムを使用し、クラウドプロバイダーはオフの
ときのみ使用されます。

## できること

- Rust 製のエージェントループを実行します。組み込みツールでファイルを読み書き・編集し、
  タスクが完了するまで進めます — 同じチャット UI・差分・「保持／取り消し」フローを
  そのまま使えます。
- **ツール:** `Read`、`Write`、`Edit`（`replace_all` 対応）、`LS`、`Grep`（`glob`/
  `case_sensitive` の絞り込み）、`Glob`、`Bash`（プロジェクト内で実行し `.venv` を有効化）。
  すべてのファイルアクセスはプロジェクトディレクトリ内に限定されます。
- **プロジェクトコンテキスト:** マスター／指示ファイル、プロジェクトマップ、インストール
  済みスキルを自動検出します（[CONTEXT_FILES.md](CONTEXT_FILES.md) を参照）。
- **メモリ:** チャットタブごとに会話を記憶します。
- **ビジョン:** 貼り付けた画像は、ビジョン対応モデル（`llava`、`llama3.2-vision` など）に
  送信されます。

## チューニング（ネイティブ有効時：設定 → プロバイダー）

- **コンテキストウィンドウ（`num_ctx`）** — モデルが一度に「見られる」量（既定 8192）。
  大きいほどメモリ／VRAM を消費します。低スペック機では下げ、長い文書／会話では上げます。
- **温度（Temperature）** — 既定 0.4（低いほど編集がより決定論的）。

## 注意・制限事項

- ツール呼び出しの品質はモデルに依存します。ごく小さい非ツールモデルより `llama3.1` /
  `qwen2.5` / `mistral-nemo` を推奨します。
- 会話メモリはプロセス内に保持されます（「新しいチャット」／タブを閉じると消去され、
  アプリ再起動をまたいだ永続化はまだ行いません）。
- 出力は応答単位です（トークン単位のストリーミングではなく、チャット UI のメッセージ
  モデルに合わせています）。
